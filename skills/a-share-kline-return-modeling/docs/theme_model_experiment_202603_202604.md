# 题材特征入模实验

## 实验设置

- 验证窗口：2026-03-02 到 2026-04-30，共 43 个交易日。
- 股票池：`00_股票清单.csv` 内 228 只股票。
- 标签目标：沿用当前主路径，训练 `future_5_return >= 20%` 爆发标签，业务评估看 `future_5_return >= 10%`。
- 输出方式：每日 Top20 候选，评估每日 Top3。
- 不改生产脚本 `01_predict_stock_direction.py` 的固定特征列表。

## 结论

题材字段作为数据层是可用的，但不能直接塞进当前第一层 LightGBM 模型。

- `add_theme_all84` 接近基线但没有超过基线：`>=10%` 从 24 降到 23，亏损从 29 升到 33。
- `add_theme_core72` 和 `replace12_theme_core60` 明显变差，说明题材热度核心字段会放大追热倾向。
- 题材特征有信息量，但当前模型没有学会区分“题材热中继续爆发”和“题材热后回落”。

## 重要性观察

`add_theme_all84` 中，题材特征占总重要性的约 26%。

最靠前的题材特征：

- `max_theme_stock_count`
- `max_theme_avg_range_pos_20_rank`
- `max_theme_avg_turnover_rank`
- `theme_count`
- `max_theme_avg_turnover_pct`
- `max_theme_avg_pct_chg`
- `max_theme_avg_range_pos_20`
- `max_theme_avg_ret_20`
- `stock_vs_theme_ret_20`

这说明模型确实在用题材字段，但用法偏向“题材热度/拥挤度”，而不是更精细的题材内个股强弱分化。

## 逐日观察

题材全量版在部分日期有改善：

- 2026-03-23：Top3 `>=10%` 从 1 提升到 2，平均收益从 6.88% 到 11.56%。
- 2026-04-10：Top3 `>=10%` 从 0 提升到 2，平均收益从 1.27% 到 6.02%。
- 2026-04-27：Top3 `>=10%` 从 1 提升到 2，平均收益从 5.59% 到 9.10%。
- 2026-04-29：Top3 `>=10%` 从 0 提升到 1，亏损从 1 降到 0，平均收益从 -2.97% 到 7.19%。

但也有明显恶化：

- 2026-03-16：亏损从 2 增到 3，平均收益从 -4.59% 降到 -14.37%。
- 2026-03-17：亏损从 1 增到 3，平均收益从 -5.29% 降到 -8.95%。
- 2026-04-15：`>=10%` 从 2 降到 1，平均收益从 11.63% 降到 4.20%。
- 2026-04-24：`>=10%` 从 2 降到 0，平均收益从 9.77% 降到 3.09%。

## 当前处理

- 保留题材清洗字段。
- 暂不把题材字段加入 `01_predict_stock_direction.py` 的生产特征列表。
- 下一步不要再简单“加特征”。更合理的方向是先学习题材状态分层：
  - 题材启动
  - 题材加速
  - 题材拥挤
  - 题材退潮

然后让个股模型在不同题材状态下学习不同形态，而不是把题材热度当普通数值特征直接混入。
