# 风险台账

更新时间：2026-05-30 GMT+8

## 用途

集中记录项目级风险、影响、Owner、应对动作和当前状态。小时报可以发现风险，但长期跟踪以本文件为准。

## 风险等级

```text
高：影响阶段验收、主线口径、可复现性或真实可交易性。
中：影响效率、解释性、局部质量或后续维护。
低：短期不阻塞，但需要定期关注。
```

## 风险更新规则

- 小时报发现新风险时，可先写入小时报；当风险会持续影响阶段验收时，补到本台账。
- 高风险必须有 Owner 和下一步动作。
- 风险关闭必须写明关闭依据，例如验收报告、测试结果、CR 审批记录或用户确认。
